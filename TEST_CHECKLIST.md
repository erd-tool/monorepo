# ERD Tool 테스트 체크리스트

현재 구현된 MVP와 observability 구성을 실제로 검증하기 위한 체크리스트입니다.

## 1. 테스트 준비

- [ ] `docker compose build` 실행
- [ ] `docker compose up -d` 실행
- [ ] `docker compose ps` 결과에서 주요 서비스가 `Up` 상태인지 확인
  - 대상: `frontend`, `gateway-service`, `auth-service`, `team-service`, `erd-service`, `collaboration`, `postgres`, `prometheus`, `grafana`, `elasticsearch`, `logstash`, `kibana`
- [ ] 브라우저에서 아래 주소 접속 확인
  - Frontend: `http://127.0.0.1:5173`
  - Gateway: `http://127.0.0.1:8080/actuator/health`
  - Prometheus: `http://127.0.0.1:9090`
  - Grafana: `http://127.0.0.1:3000`
  - Kibana: `http://127.0.0.1:5601`

## 2. 회원 기능

- [ ] 로그인 페이지 진입
  - 기대 결과: 로그인/회원가입 탭과 아이디, 비밀번호 입력 폼이 노출된다.
- [ ] 회원가입 성공
  - 입력 예시: `loginId`, `displayName`, `email`, `password`
  - 기대 결과: 가입 후 자동 로그인되거나 정상 세션이 생성된다.
- [ ] 로그인 성공
  - 기대 결과: `/app` 대시보드로 이동한다.
- [ ] 잘못된 비밀번호로 로그인 실패
  - 기대 결과: 사용자 친화적인 오류 메시지가 표시된다.
- [ ] 로그아웃
  - 기대 결과: 로그인 화면으로 돌아간다.

## 6. 엔티티 기능

- [ ] 좌측 `엔티티 생성` 버튼 클릭
  - 기대 결과: 새 엔티티가 캔버스에 생성된다.
- [ ] 엔티티 편집기 열기
  - 기대 결과: 엔티티명, 설명, 컬럼 목록, 색상 선택 UI가 표시된다.
- [ ] 엔티티 논리/물리/논리·물리 모드 전환
  - 기대 결과: 캔버스와 편집기 항목 표시 형식이 모드에 맞게 달라진다.
- [ ] 엔티티 배경색 10종 변경
  - 기대 결과: 파스텔톤 단색 배경이 즉시 반영된다.
- [ ] 엔티티 설명 입력
  - 기대 결과: 저장 후 유지된다.
- [ ] 컬럼 추가
  - 기대 결과: 새 컬럼 행이 생성된다.
- [ ] 컬럼명, 속성명, 자료형, 기본값, 코멘트 수정
  - 기대 결과: 입력 UI가 넘치지 않고 저장된다.
- [ ] PK 체크
  - 기대 결과: PK 컬럼이 엔티티 상단으로 이동한다.
- [ ] PK 2개 이상 체크
  - 기대 결과: PK로 설정한 순서대로 상단에 정렬된다.
- [ ] 컬럼 삭제
  - 기대 결과: 해당 컬럼이 제거된다.
- [ ] 엔티티 삭제
  - 기대 결과: 캔버스와 문서 데이터에서 제거된다.

## 7. 관계 기능

- [ ] 좌측 `관계 설정` 버튼 클릭
  - 기대 결과: 관계 설정 모드가 활성화된다.
- [ ] 부모 엔티티 클릭 후 자식 엔티티 클릭
  - 기대 결과: 좌상단에 관계 옵션 팝업이 열린다.
- [ ] 관계 옵션이 `radio` 형식인지 확인
  - 항목: 식별/비식별, 카디널리티, 필수/선택
- [ ] 관계 생성 완료
  - 기대 결과: IE 표기법 관계선이 생성된다.
- [ ] 관계 생성 시 자식 엔티티에 FK 컬럼 자동 추가
  - 기대 결과: 부모 PK 이름/자료형과 동일한 FK 컬럼이 생성된다.
- [ ] 동일 엔티티 사이에 여러 관계 생성
  - 기대 결과: 관계선이 완전히 겹치지 않고 구분 가능하다.
- [ ] 관계선 드래그 조정
  - 기대 결과: 엔티티 연결은 유지하면서 선 경로만 유연하게 조정된다.
- [ ] 관계 편집기에서 설명 수정
  - 기대 결과: 설명 영역이 2줄 크기로 표시되고 저장된다.
- [ ] 관계 삭제
  - 기대 결과: 선이 삭제되고 자식 엔티티의 FK 컬럼도 함께 제거된다.
